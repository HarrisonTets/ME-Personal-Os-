# Build Log · Session 07 · 2026-06-20

> **Goal of this session:** Add **Daily Reflections** — the evening check-in
> (mood, energy, what drained/energized you). This is the "learns from you" loop
> from the spec (§6.1.4): the self-awareness data that will later shape your
> pillar scores and suggestions.

## What we built

### Daily reflection — reachable from Today
We didn't add a 6th tab (five is the sensible maximum for a bottom bar). Instead,
the **Today** screen now has an **Evening reflection** card near the top:
- Not done yet → "How did today feel? Take a moment."
- Done → shows your mood + energy and "tap to edit".

Tapping it opens the reflection screen.

### The reflection flow (full slice)

**The brain (domain):**
- New entity `Reflection` (mood + energy on a shared low/medium/high `Scale`,
  plus two optional notes), one per day.
- New storage promise `ReflectionRepository`.
- New rule `SaveReflection` — generates the id, normalises the date, trims empty
  notes to null.

**The memory (data):**
- New database table `Reflections`.
- **Migration v6 → v7** (your existing data preserved).
- `ReflectionRepositoryImpl` enforces **one reflection per day** (saving replaces
  any existing entry for the day).

**The face (presentation):**
- **Daily reflection screen**: tap-to-pick **Mood** (😔/😐/😊) and **Energy**
  (🪫/🔋/⚡), two prompts ("What drained you?", "What energized you?"), and a
  **Past reflections** history list below.
- If today's reflection already exists, the form **pre-fills** with it, so the
  same screen works for both creating and editing.
- **Today card** entry point (`ReflectionTodayCard`).

## How it behaves

1. On **Today**, tap **Evening reflection**.
2. Pick a mood and an energy level, optionally jot what drained/energized you.
3. **Save** → back on Today, the card now shows "Reflection done · Mood … ·
   Energy …".
4. Tap it again to review or edit; scroll down to see past days.

## Design note: not yet wired into suggestions

The spec says reflections eventually **feed pillar scores and suggestion weights**
(e.g. a run of low-energy evenings → lighter suggestions next day). We're
**storing** that data now but haven't changed the suggestion engine yet — that's
a deliberate next step, noted so it isn't forgotten. (The engine already accepts
an energy signal, so this will slot in cleanly.)
