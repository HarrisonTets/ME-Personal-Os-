# Build Log · Session 10 · 2026-06-20

> **Goal of this session:** Add a **Settings screen** so the choices made during
> onboarding (pillars, daily budget, reminder intensity) can be changed anytime.

## What we built

### A Settings screen
Reachable from a **⚙️ gear icon** on the Today screen (top-right). It shows:

- **Focus pillars** — a switch per pillar; turn areas on/off. Toggling a pillar
  immediately enables/disables that pillar's habits (so Today adjusts). We guard
  against turning **all** of them off (you must keep at least one).
- **Daily actions** — pick 3 / 4 / 5; instantly changes how many suggestions
  Today shows.
- **Reminder intensity** — Low / Medium / High.

Every change **saves immediately** (no "Save" button) and the rest of the app
reacts straight away, because everything reads from the same settings state.

### Controller updates
`SettingsController` gained `updateDailyBudget`, `updateReminderIntensity`, and
`updateEnabledPillars` (plus a `copyWith` on the state to keep them tidy). These
mirror what onboarding does, just one setting at a time.

## How it behaves

1. On **Today**, tap the **⚙️** top-right.
2. Flip off a pillar → its habits stop being suggested; flip it back on → they
   return.
3. Change the daily number → go back to Today → the list length adjusts.

## A small Flutter detail we handled

Flutter recently changed how radio buttons work: the old per-item `groupValue` /
`onChanged` on `RadioListTile` are deprecated. We used the **new `RadioGroup`**
wrapper (the modern, recommended API) for the reminder-intensity choice — so the
code is current, not using anything on its way out.

## Why this matters

the app from "fixed" into something the user genuinely controls — and the
settings store now has a real screen behind it, ready to hold more options
(currency, targets, theme) as we add them.
