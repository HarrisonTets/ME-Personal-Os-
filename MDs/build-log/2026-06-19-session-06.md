# Build Log · Session 06 · 2026-06-19

> **Goal of this session:** Connect the pillars to the **Today** screen, so doing
> an activity in any pillar automatically ticks its matching suggestion. This is
> what makes the five pillars feel like one connected system.

## The idea

Until now, each pillar was its own island. You could finish a Bible reading on
the Bible tab, but the "Read today's passage" suggestion on **Today** stayed
unticked — you'd have to mark it done twice.

Now they're linked. Do the activity once, and Today updates itself.

## How it works (the mechanism)

We gave each habit an optional **trigger key** — a label naming the activity that
completes it. When you perform that activity, any habit with the matching trigger
is marked done for the day.

The links we wired:

| Do this… | …auto-ticks this habit on Today |
|----------|--------------------------------|
| Finish a **Quiet Time** reading | "Read today's passage & reflect" |
| Add a **Finance** entry | "Log yesterday's expenses" |
| Complete a **Focus** session (timer runs out) | "One deep focus session" |
| Log **water** (Health) | "Drink a glass of water" |
| Log a **walk** (Health) | "Take a 10-minute walk" |

(The trigger keys live in one place — `HabitTriggers` — so the pillars and the
seed can't disagree on the spelling.)

## What we changed

**The brain (domain):**
- Added an optional `triggerKey` to the `Habit` entity.
- New use case `AutoCompleteHabits` — given a trigger key, it marks every active
  habit with that key done for today. It's **idempotent** (safe to run twice).
- The four pillar use cases (`CompleteQuietTime`, `AddFinanceEntry`,
  `AddFocusSession`, `AddHealthLog`) now call `AutoCompleteHabits` after saving.
  - Focus only triggers when the session actually **completed** (not ended early).
  - Health maps water→water-habit and walk→walk-habit.

**The memory (data):**
- Added a `triggerKey` column to the Habits table.
- **Migration v5 → v6**: adds the column, then **backfills** your existing default
  habits by name (so the feature works on your current data without re-seeding).
- The seed now sets trigger keys for fresh installs.

**Why it "just works" on Today:** the Today screen already watches the habit-log
data live. When `AutoCompleteHabits` writes a "done" log, Today re-runs its
suggestion engine and the item slides into "Completed" — no extra wiring needed.
This is the payoff of the clean architecture we set up in session 01.

## How it behaves

1. Go to the **Finance** tab, add an expense.
2. Switch to **Today** → "Log yesterday's expenses" is already ticked. ✅
3. Run a **Focus** session to completion → "One deep focus session" ticks itself.
4. Same for a Bible reading, logging water, or a walk.
