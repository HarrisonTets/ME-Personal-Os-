# Build Log · Session 01 · 2026-06-19

> **Goal of this session:** Set up the project and build the very first working
> piece end-to-end — the **Today screen** that suggests daily actions.

## What we decided first

Before writing code, we locked three foundational choices (details in
[`decisions/`](../decisions/)):

1. **Riverpod** for state management (data ↔ screen syncing).
2. **Drift + SQLite** for storing data on the phone.
3. **"Vertical slice first"** — build ONE complete feature through all layers to
   prove the structure works, before spreading out to every pillar.

## What we actually built

### 1. Created the Flutter project
- Generated the project files, targeting **Android** (primary) and **Windows**.
- Gave it its own clean **git repository** (version history) — previously it was
  tangled in a repo covering your whole user folder.

### 2. Added our "ingredients" (packages)
Added Riverpod, Drift, SQLite, plus small helpers (`uuid` for IDs, `intl` for
dates). These went into `pubspec.yaml`.

### 3. Built the brain (Domain layer)
- Defined the core nouns: `Pillar`, `Habit`, `HabitLog`, `Suggestion`.
- Wrote the **suggestion engine** (`GenerateTodaySuggestions`) — the logic that
  picks your 3–5 daily actions. It ranks habits by:
  - **Weaker pillars first** (areas you've neglected this week float up),
  - **Protecting streaks** (don't let a good run break),
  - **Resurfacing** habits you haven't done in a while,
  - **Favoring lighter tasks** to help you build momentum,
  - then caps the list to a daily budget with a per-pillar limit so it stays
    small and balanced.

### 4. Built the memory (Data layer)
- Defined two database tables: `Habits` and `HabitLogs`.
- Ran **code generation** to create the database plumbing automatically.
- Wrote the **repository** that saves/loads using Drift.
- Wrote a **seeder** that adds 6 sensible starter habits the first time you open
  the app (one per pillar, plus a hydration one).

### 5. Built the face (Presentation layer)
- Wired everything together with Riverpod providers.
- Built the **Today screen**: a date header, a progress bar, a list of suggested
  action cards (each with pillar color/icon, time estimate, and a reason), and a
  "Completed" section.
- Made it **live**: tick a habit and it instantly moves to Completed and updates
  progress — no manual refresh.

## How we checked it works ✅

## One bump we hit (and fixed)

The first Android build **failed on a network timeout** while downloading Gradle
(Android's build tool) — nothing wrong with our code. We switched to a smaller
download and it built fine on the retry.

> **New term:** *Gradle* is the tool Android uses to assemble the app. The first
> build downloads it, which is slow on a weak connection but only happens once.

## Where things stand

The skeleton + first feature are done and verified. The app currently:
- Opens to the Today screen,
- Shows a balanced set of suggested actions across the 5 pillars,
- Lets you tick them off with live progress.
