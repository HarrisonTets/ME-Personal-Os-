# Build Log · Session 03 · 2026-06-19

> **Goal of this session:** Build the **Finance pillar** — a new tab to log
> income/expenses in a few taps and see this week's totals.

## What we built

### A new "Finance" tab
The bottom bar now has three tabs: **Today · Bible · Finance**.

### The Finance pillar (full slice)

**The brain (domain):**
- New entity `FinanceEntry` (an income or expense: amount, category, date, note),
  with a fixed set of categories (Food, Rent, Transport, Giving, Business, Other).
- New storage promise `FinanceRepository`.
- Two rules:
  - `AddFinanceEntry` — creates and saves an entry (generates its id).
  - `SummarizeSpending` — a **pure, tested** function that rolls a list of
    entries into totals + the top spending categories (just like the suggestion
    engine, it's pure so we can unit-test it easily).

**The memory (data):**
- New database table `FinanceEntries`.
- **Migration v2 → v3**: existing installs gain the finance table without losing
  anything. (Each new version adds one `if (from < N)` block — see
  [concepts/06-navigation-and-migrations.md](../concepts/06-navigation-and-migrations.md).)
- `FinanceRepositoryImpl` to read/write it.

**The face (presentation):**
- **Finance home**: a "This week" card (spent, income, entry count), a **Top
  categories** bar chart, and a list of this week's entries.
- **Quick-Add sheet**: slides up from the bottom; pick Expense/Income, type an
  amount, tap a category chip, optional note, Save. A few taps, done.

### A small shared helper
- `lib/core/money.dart` formats amounts as currency. Default symbol is **"K"
  (Zambian Kwacha)** — change that one line to use another currency, or we can
  make it a user setting later.

## How it behaves

1. Open the **Finance** tab → "This week" shows K 0.00 spent (empty to start).
2. Tap **Add** → the Quick-Add sheet appears.
3. Enter e.g. amount `150`, category **Food**, Save.
4. The sheet closes; the week total jumps to "K 150.00 spent", Food appears under
   Top categories, and the entry shows in the list.
5. "This week" means **since Monday** — entries older than that roll off.

ideas you'll see reused: a **bottom-sheet form** (Quick-Add) and a **pure
calculation function** with its own tests (`SummarizeSpending`).
